Golang

包管理

Go语言的源码复用建立在包(package)基础之上，包通过package,import,GOPATH操作完成

main包

Go语言的入口main()函数所在的包(package)叫main，main包想要引用别的代码,需要import导入

package

src目录是以代码包的形式组织并保存Go源码文件的。

同一个目录下的源码文件第一行声明的所属包，必须一致

同一个目录下的所有.go文件的第一行添加包的定义,以标记该文件归属的包.

包需要满足:

  • 一个目录下的同级文件归属一个包。
  • 在同一个包下的文件package名建议都是该目录名,但也可以不是
  • 包名为main的包为应用程序的入口包，其他包不能调用

GOPATH环境变量

import 导入时，会从GO的安装目录(GOROOT环境变量设置的目录)和GOPATH环境变量设置的目录中检索src/package来导入包，不存在则导入失败。GOROOT/GOPAHT可以修改。

  • GOROOT：GO内置的包所在的位置
  • GOPATH：自己定义的包的位置

import特殊用法

.代表导入这个包的函数时，可以省略前缀包名。

起别名。

如果仅仅需要导入包时执行初始化操作，并不需要使用包内的其他函数，常量等资源，则可以在导入包时，匿名导入

init函数

init()函数和main()函数是go语言中的保留函数。如果包中存在init()函数，那么init()中的代码会在main()函数执行前执行，用于初始化包所需要的特定资料。

init()和main()区别

  • 相同点:两个函数在定义时不能有任何的参数和返回值。该函数只能由go程序自动调用,不可以被引用。
  • 不同点：
    • init()函数可以应用于任意包中，且可以重复定义多个
    • main()函数只能用于main包中，且只能定义一个

两个函数的执行顺序

  • 在main包中的go文件默认总是会被执行的
  • 对于一个go文件的init()调用顺序是从上到下的
  • 对同一个package中的不同文件，将文件名按字符串进行“从小到大”排序，之后顺序调用各文件中的init()函数
  • 对于不同的package，如果不相互依赖的话，按照main包中的import的顺序调用其包中的init()函数
  • 如果package存在依赖，调用顺序为最后被依赖的最先被初始化，例如:导入main->A->B->C，则初始化顺序为C->B->A->main。
